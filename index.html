Rapid Tileset Designer

This game is actually not a game, but a tool for rapidly envisioning a tileset in a level. The real functionality is what's under the hood; I've created an intuitive, fast workflow for desigining the visual aesthetics of a tilesheet and tile-based level. The p5 canvas displays how the level would look through a game camera, and the white square (representing the player) can be moved with WASD controls.

Above the game screen, the tile sheet is displayed. You can left-click to draw on the tile sheet, and left-click while holding Alt to pick a drawing color from the mouse position. The level will be updated in realtime.

How it works:

As the designer, you only need to provide two images: a Tile Sheet and a Level Map.

At the bottom of the tile sheet, there's a key with colors that correspond to the tiles.

The colors on the sheet show the program how to map the tiles on the level map. Level layouts can be designed pretty quickly because the maps are actually very tiny; each tile is represented by one pixel.

Following the same format as the level map, you can also provide a collision map, which tells the canvas where your player collides with walls and other solid objects.

If I expand on this concept in the future, I'd like to add the ability to edit the tile map in-browser (right now, you can only edit it locally)
